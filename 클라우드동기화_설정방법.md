# ☁️ 클라우드 동기화 설정 방법 (Supabase)

> 이 단계를 마치면 **PC와 폰에서 같은 데이터**가 자동으로 동기화돼요.
> 키를 넣기 전까지는 스케쥴러가 **오프라인 모드**로 잘 작동합니다. 천천히 하셔도 돼요.

---

## 1단계 — Supabase 프로젝트 만들기 (무료)

1. https://supabase.com 접속 → **Start your project** → 가입 (GitHub 또는 이메일)
2. **New project** 클릭
3. 입력:
   - **Name**: 아무거나 (예: `my-scheduler`)
   - **Database Password**: 아무거나 정하고 **따로 메모** (나중에 직접 쓸 일은 거의 없음)
   - **Region**: `Northeast Asia (Seoul)` 추천
4. **Create new project** → 1~2분 기다리면 생성 완료

---

## 2단계 — 데이터 저장 테이블 만들기

1. 왼쪽 메뉴에서 **SQL Editor** (📄 아이콘) 클릭
2. **New query** → 아래 내용 그대로 붙여넣고 **Run** (▶) 실행

```sql
-- 사용자별 스케쥴 데이터 테이블
create table scheduler_data (
  user_id uuid primary key references auth.users on delete cascade,
  data jsonb not null default '{}'::jsonb,
  updated_at timestamptz default now()
);

-- 보안: 본인 데이터만 접근 가능하게
alter table scheduler_data enable row level security;

create policy "own data only" on scheduler_data
  for all
  using (auth.uid() = user_id)
  with check (auth.uid() = user_id);

-- 실시간 동기화 켜기
alter publication supabase_realtime add table scheduler_data;
```

> "Success. No rows returned" 이 뜨면 정상이에요.

---

## 3단계 — (선택) 이메일 인증 끄기 = 가입 즉시 로그인

기본값은 가입 시 **확인 메일**을 거쳐야 해요. 혼자 쓰는 거라 번거로우면 끄세요.

1. 왼쪽 메뉴 **Authentication** → **Sign In / Providers** (또는 **Providers**)
2. **Email** 항목에서 **Confirm email** 을 **끄기(off)**
3. 저장

> 이걸 끄면 회원가입 → 바로 로그인 가능. (켜두면 메일 인증 후 로그인)

---

## 4단계 — 키 2개 복사해서 붙여넣기

1. 왼쪽 메뉴 맨 아래 **Project Settings (⚙️)** → **API**
2. 두 값을 복사:
   - **Project URL** (예: `https://abcd1234.supabase.co`)
   - **anon public** key (`eyJhbGci...` 로 시작하는 긴 문자열)
3. `스케쥴러.html` 파일을 메모장(또는 편집기)으로 열고, 위쪽의 이 부분을 찾아 채워요:

```js
const SUPABASE_URL='';        // ← 여기에 Project URL
const SUPABASE_ANON_KEY='';   // ← 여기에 anon public key
```

채운 예시:
```js
const SUPABASE_URL='https://abcd1234.supabase.co';
const SUPABASE_ANON_KEY='eyJhbGciOiJIUzI1NiIsInR5cCI6...';
```

4. 저장 후 브라우저에서 **새로고침**

> ⚠️ anon 키는 외부에 공개돼도 안전한 키예요 (위 2단계의 보안 정책이 데이터를 지켜줍니다).
> 단, `service_role` 키는 절대 넣지 마세요.

---

## 5단계 — 사용하기

1. 우측 상단 **☁️ 로그인** 버튼 → **회원가입** (이메일 + 비밀번호 6자 이상)
2. 로그인하면 버튼이 **✅ 동기화 중** 으로 바뀜
3. **폰에서도** 같은 `스케쥴러.html`을 열고 **같은 계정으로 로그인** → 자동으로 같은 데이터!

---

## 📱 폰에서 여는 방법 (참고)

폰은 바탕화면 파일을 직접 못 여니, 아래 중 하나로 옮기면 돼요:
- **간단**: 파일을 **카카오톡 '나에게 보내기'** 나 이메일로 보내서 폰에서 열기
- **추천**: 무료 호스팅에 올려 주소로 접속 (예: Netlify Drop — `스케쥴러.html`을 끌어다 놓으면 URL 생성)
  - 이러면 PC·폰 모두 같은 주소로 접속 → 제일 편함. 원하면 이 방법도 도와드릴게요.

---

## 문제가 생기면

- 로그인 시 "Email not confirmed" → 3단계로 Confirm email 끄기, 또는 받은 메일 인증
- ☁️ 버튼이 **오프라인** 그대로 → 키가 비었거나 오타. 4단계 다시 확인 + 새로고침
- 동기화가 안 됨 → 2단계 SQL을 실행했는지, 두 기기가 **같은 계정**인지 확인
